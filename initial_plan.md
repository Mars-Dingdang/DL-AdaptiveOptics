为了保证你们能顺利推进并拿高分，我总结了以下**三大核心难点**以及对应的**破局策略**，还有项目执行中的**注意事项**：

---

### 难点一：生成模型的“幻觉（Hallucination）”与科学真实性的冲突（最致命）
*   **问题所在**：GAN 或 Diffusion 这种生成模型（AIGC）有一个通病——**喜欢“脑补”细节**。在日常图片去噪（比如修人像）时，脑补一点皮肤纹理没关系；但在**遥感图像**中，这可能是致命的。如果模型把一个模糊的军事设施“脑补”成了一栋民房，或者把模糊的农田纹理“生成”了不存在的公路，这就失去了“AI for Science”的意义。
*   **破局策略**：
    *   **损失函数设计（Loss Function）**：不能仅仅依赖对抗损失（Adversarial Loss），必须引入**强烈的结构约束**。在你的总 Loss 中，加大 L1/L2 像素级损失和 SSIM（结构相似度）损失的权重，压制模型过度自由发挥。
    *   **引入物理先验**：如果在模型结构中能把退化过程（Blur Kernel / Noise）作为条件输入，或者利用频域（傅里叶变换）信息来约束生成过程，会极大提升真实性，这也是**写 Report 时最能体现创新点**的地方。

### 难点二：成对数据集的构建与物理退化模拟（最花时间）
*   **问题所在**：正如之前提到的，你很难拿到同一时间、同一地点“受大气湍流影响的模糊图”和“没有湍流的高清图”。如果只用普通的“高斯模糊”来模拟，老师会觉得太简单，无法体现“自适应光学”的物理背景。
*   **破局策略**：
    *   **查阅开源的退化模拟器**：去 GitHub 搜一下别人写好的大气湍流模拟代码（关键词：*Atmospheric turbulence simulator, Zernike polynomials phase screen*）。利用光学中的“泽尼克多项式”或“相位屏”理论来生成畸变。
    *   **降维打击**：如果物理模拟太难，建议采用类似 **Real-ESRGAN 的高阶退化模型（High-order Degradation）**——即将不同程度的模糊、噪声、压缩伪影进行随机组合。这样虽然不是严格的“大气湍流”，但极大地逼近了真实的复杂退化场景。在 Proposal 里可以包装为“复杂气象与传感器退化综合模拟”。

### 难点三：计算资源与 Live Demo 的实时性（最影响展示效果）
*   **问题所在**：遥感图像通常分辨率极高（比如 4K 或 8K）。如果你用 Diffusion Model，推理一张图可能需要几十秒甚至几分钟。在 Poster Session 现场，几十秒的等待时间会让看展的老师和同学感到非常尴尬，展示效果大打折扣。
*   **破局策略**：
    *   **切片处理（Patch-based）**：训练和演示时，只处理 256x256 或 512x512 的图像块。
    *   **模型选择的妥协**：对于 Live Demo，**GAN（如 Pix2PixHD, ESRGAN 的变体）由于是一次前向传播，速度极快（几百毫秒），是做 Demo 的首选**。如果你执意要用 Diffusion，必须使用加速采样算法（如 DDIM, DPM-Solver 或最近的 LCM），把推理步数压到 4-10 步以内。
    *   **“伪”Live Demo 后备方案**：现场带两套方案。一套是真正的实时小图推理；另一套是预先处理好几张 4K 大图，用一个可以左右滑动的 UI 组件（Image Slider）让大家对比“Before / After”，这在视觉上同样极具冲击力。

### 项目执行中的关键注意事项 (Precautions)

1.  **切忌“好高骛远”，一定要先跑通 Baseline**
    *   **第一周一定要做的事**：不要一上来就搞最复杂的 Latent Diffusion 或 Transformer 架构。先用最经典的 **U-Net** 或者基础的 **Pix2Pix** 跑一个版本。
    *   **为什么？** 哪怕最后高级模型训练失败或者算力不够，你们至少有一个能出图的保底模型，不至于期末挂科。有了 Baseline 后再迭代复杂模型，Report 里也就有了对比的表格（Baseline vs. Proposed Method），这是高分的必需品。

2.  **评价指标（Metrics）不能只看一个**
    *   生成式图像恢复的评价在学术界一直是个难点。你需要同时汇报两类指标：
        *   **客观失真指标**：PSNR, SSIM（越高越好，代表与原图有多像）。
        *   **主观感知指标**：LPIPS, FID（越低越好，代表看起来有多清晰、多像真实图像）。
    *   *注意：生成模型通常 LPIPS 很好，但 PSNR 偏低；传统模型 PSNR 高，但看起来模糊。在 Report 中讨论这种现象，会显得你们对深度学习的理解非常深刻。*

3.  **UI 界面（前端）非常重要**
    *   因为展示占了足足 **15 分（Poster + 现场）**。一个简陋的命令行黑框跑出来的结果，跟一个有着网页界面的结果，在助教和路人眼里的主观评分差距极大。
    *   **强烈建议团队中的同学 C 花两三天时间学一下 `Gradio` 或 `Streamlit`**。用几十行 Python 代码就能写出一个带“上传图片”、“加载进度条”和“左右滑动对比图”的精美网页，这绝对是拿 Bonus 的大杀器。

4.  **与助教（TA）保持沟通（极其重要）**
    *   你们的项目属于“AI for Science”和“AIGC”的交叉，在 4 月 20 日提交 Proposal 前，**务必带着你们找好的“数据集”和“退化生成方案”找助教聊一次**。
