## Plan 0328: 独立 Val/Test 协议落地

目标是在不改模型结构的前提下，把训练调参与最终报告评估解耦：验证集用于调参，测试集仅用于最终一次性报告，避免验证集过拟合。

### 1. 配置层定义数据协议
1. 在 configs/default.yaml 的 data 段新增 test_root，并保留 val_root 与 val_ratio。
2. 约束规则：
   - train.py 训练只读取 train_root 与 val_root。
   - 当 val_root 为空时，继续使用 train_root + val_ratio 自动切分验证集。
   - test_root 不能为空时，专用于最终评估，不参与训练和调参。
3. 在 README 里补充推荐目录协议与命名约定，明确 val/test 的职责边界。

### 2. 训练阶段行为固定
1. 复用 train.py 现有逻辑：
   - 若 val_root 非空，直接使用独立验证目录。
   - 若 val_root 为空，才使用随机切分。
2. 增加启动前配置校验策略：
   - 当 train_root 与 val_root 指向同一目录且 val_root 非空时给出警告。
   - 当 test_root 与 train_root/val_root 重叠时给出警告。
3. 明确实验流程：仅根据验证集曲线选择 best checkpoint。

### 3. 评估脚本支持 test split
1. 在 eval.py 新增 split 参数（val/test）。
2. split=val：保持现有逻辑（val_root 或 train_root 切分）。
3. split=test：只读取 test_root；若为空则直接报错并退出。
4. 评估报告中写入 split 字段，避免后续混淆 val 指标与 test 指标。

### 4. 执行与冻结策略
1. 训练迭代期间反复运行 split=val 评估。
2. 选定最终模型后，仅运行一次 split=test 生成最终结果。
3. 报告中同时展示 val 与 test，但以 test 作为最终结论。

### 5. 验证清单
1. 用独立 val_root + test_root 启动训练，确认训练仅访问 train/val。
2. 运行 eval.py split=val 与 split=test，检查输出目录和 metrics 字段可区分。
3. 人工抽查样本，确认 test 集图片未出现在训练与验证目录。
4. 固定 seed 重跑一次，验证 val 指标可复现，test 结果仅在最终阶段生成。
